## 2. 软件过程模型

> 常考选择题，重点理解各种模型的优缺点和适用场景

### 传统模型
- [[瀑布模型]] — 文档驱动，适合需求明确的项目
- [[增量模型]] — 分批交付，逐步完善
- [[V模型]] — 测试与开发阶段对应
- [[原型模型]] — 快速原型，适合需求不明确
- [[螺旋模型]] — 风险驱动，大型项目

### 迭代与统一过程
- [[迭代模型]] — 每次迭代一个完整增量
- [[RUP]] — 用例驱动、架构为中心、迭代增量
  - 四个阶段：初始 → 细化 → 构造 → 移交
  - 五个核心工作流

## 4. 软件设计

> **核心重点**，设计原则和结构化设计常考

### 设计基础
- [[软件设计原则]] — 模块化、抽象、信息隐藏、独立性
- [[内聚与耦合]] ⭐ — 高频考点
  - 内聚（由低到高）：偶然 → 逻辑 → 时间 → 过程 → 通信 → 顺序 → 功能

- [[模块化设计]] — 模块的扇入扇出

  - 功能性、可靠性、易用性、效率、维护性、可移植性、兼容性、安全性
- [[McCall 质量模型]]

### 质量保证
- [[SQA 软件质量保证]]
- [[软件评审]] — 同行评审、走查 Walkthrough、审查 Inspection
- [[软件度量]]
  - 面向规模的度量（LOC/KLOC）
  - 面向功能点的度量（FP）
  - 面向人的度量（人月）
- [[软件可靠性]]
  - MTBF（平均无故障时间）
